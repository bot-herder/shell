Detailed Description

Describes a token from the input.

Which kinds of tokens will be recognized can be choosen with setLevel(). By default the most advanced BASH/KSH set is recognized.

enum Token::Level

The constants that determine which tokens will be recognized by the tokenizer.

Enumerator:
BASIC 

Basic tokens: ; | < > >> &.

ADVANCED 

Advanced = BASIC plus: && || ( ) shell-variables simple-wildcard.

SH 

Shell scripts = ADVANCED plus: << ` ;; { }.

BASH 

Bash/ksh scripts = SH plus: $( )

enum Token::token_t

The different kinds of tokens.

Enumerator:
END 

end-of-input.

Note:
We use END here because the code EOF is a macro already defined in some other include file.
EOL 

end-of-line (either \n or \r)

SEQUENCE 

; e.g. foo ; bar

PIPE 

| e.g. run | in | tandem

INPUT 

< e.g. input < from ...

OUTPUT 

> e.g. output > to ....

APPEND 

>> e.g. output >> after ...

BACKGROUND 

& i.e. don't wait

AND 

&& in a conditional execution.

OR 

|| in a conditional execution.

SUBS 

( subshell begin, as in: ( ... ; ... )

ENDS 

) subshell end, as in: ( ... ; ... )

VAR 

$word or one of $* $@ $# $0...$9 $? $! $$

Note:
The token-text does NOT contain the $ character.
EXPR 

A simple wildcard containing * or ?

Note:
The [...] wildcard is NOT recognized.
HERE 

<< start of an inline document

BQUOTE 

` as in ` ... ` run a subshell & substitute the output here

BREAK 

;; as in: case ... in ... ;; ... ;; ... esac

GROUP 

{ group begin, as in: { ... ; ... }

GRPEND 

} group end, as in: { ... ; ... }

EVAL 

$( ... ) run a subshell & substitute the output here

WORD 

anything else.

Note:
Special words such as 'exit', 'cd' etc. are NOT handled here, because their meaning is position dependent.
