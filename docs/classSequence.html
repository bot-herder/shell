Detailed Description

A sequence of pipelines that are to be executed one after the other.

For instance: 

     abc ; def ; ghi & jkl ; ....

If the ';' separator is used (e.g. "abc") the shell will wait for the pipeline to finish before proceeding to the next pipeline.

If the '&' separator is used, the preceding pipeline (i.e. "ghi") becomes a so called 'background' pipeline. The shell will not wait for it's completion but immediately proceed to the next pipeline.

Background pipelines run at a lower priority, i.e. will have a higher 'nice(2)' value (usually +5). Their standard input will be connected to "/dev/null" and they have the INTERRUPT and QUIT signals ignored.
